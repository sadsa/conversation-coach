From rounded bubbles to full-width rows

Today every list item is a detached rounded-xl border card floating on the cream canvas with an 8px gap between rows. It reads tidy but wastes horizontal space and fragments the list into competing tiles.

These mockups keep the brand intact — warm canvas, large readable type, scannable states, the row ··· menu — but let rows run the full width of the column. The column itself stays max-w-2xl; "full width" means each row bleeds to the column's gutter edge (dashed violet guides) instead of sitting inside its own bordered box. Shown on both list surfaces: SessionList (Review) and WriteList (Study).

Implementation notes

Full-bleed
Inside the max-w-2xl px-6 md:px-10 column, a row becomes edge-to-edge with -mx-6 md:-mx-10 and re-adds the gutter as its own padding (px-6 md:px-10). Hover fill and dividers then touch the column edge; text stays aligned.
States
Status colour (ready / analysing spinner / error) and the muted archive treatment all survive — they live in the row body, not the card chrome.
··· menu
Proposals (01–03) swap the vertical ⋮ for a horizontal ⋯ pinned to the row's top-most line instead of vertically centred — so tall flashcard rows keep the control anchored to the title, matching the reference card. Still absolute against the <li>, faint on mobile, hover-revealed on desktop; swap the icon name more-vertical → more-horizontal in RowActionsMenu and change its wrapper from inset-y-0 items-center to a top offset.
Touch target
Row vertical padding bumped to ~18px so the full-width hit area clears 44px without the card border to imply edges.
Scope
One change to the row wrapper in SessionList.tsx + WriteList.tsx; list space-y-2 drops to a shared bordered <ul>.
