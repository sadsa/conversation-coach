Practice · idle screen redesign

Two ways to start: a casual chat, or pick up a call.

Today the practice page opens with a single "Start chatting" button. Real users freeze — they don't know what to talk about. This concept adds a second mode, Pick up a call, where Claude generates a fresh character for every session: someone calls in, introduces themselves, and the conversation develops naturally from whatever they bring. The familiar casual chat stays for days when you just want to talk.

Refinements from feedback: Layout B is the pick. "Stranger" is gone — too cold. "Open chat" is now "Casual chat". The CTA is "Pick up". A discreet "Try another line" control lives inside the call for when the situation isn't landing or the vocab feels out of reach.

Idle screen

Two equal-weight cards. Pick up a call is visually weighted with the call-green palette so it reads as the moment of arrival; casual chat sits quietly in surface neutrals. Order matters — the new mode goes first so it's discoverable, but neither dominates.

Short ringing beat while Claude generates the character (~1.5s), then the coach opens in role. If the situation isn't landing — too tricky, wrong vibe — a discreet Try another line control swaps in a fresh caller without ending the session.

Sample personas (what Claude might generate)

Each generation samples across orthogonal axes — identity, reason for calling, mood, location, hook. Combinatorial freshness. With "anything goes" tone the pool ranges from real-world (kiosquero, ex-compañera de tango) to absurd (time-travelling milonguero, parrot's owner).

Asks the user for advice. Conversation drifts toward weekend plans, travel disasters, calling family.

Forces narrative reaction (¿en serio?, ¿y entonces?). Coach reveals the gossip in beats — user has to react and ask.

Plan-making vocab + a tangent the user can follow or steer. Builds shared "history" the user has to play along with.

Built-in escape hatch (it's a wrong number) — user can stay or disengage. Excellent for low-confidence days.

How it plumbs in

Small change. One new route, one new option on connect(), one new state in PracticeClient. Reroll reuses the same persona route.

PracticeClient.startCall() └─ GET /api/practice/persona ──→ Claude (haiku, ~1.5s, ~$0.001) Returns { name, opener, systemPromptAddendum } └─ setState('ringing') ──→ pulse animation + "Someone's calling" └─ connect(targetLanguage, callbacks, { systemPrompt: base + persona.systemPromptAddendum, openingLine: persona.opener, ← new option transcription: true, }) └─ Gemini speaks persona.opener first └─ setState('active') ──→ fade ringing → transcript view PracticeClient.tryAnotherLine() ← user taps the reroll pill └─ agent.disconnect() └─ GET /api/practice/persona ──→ fresh persona └─ insert "New caller" divider in liveTurns (visually only — no DB write) └─ dim previous turns (CSS .stale class) └─ connect(..., { openingLine: newPersona.opener }) └─ Timer continues — same 5-min budget; reroll doesn't refund time Idle screen state (persisted in localStorage): mode: 'chat' | 'call' ← last choice remembered
