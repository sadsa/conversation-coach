P1 · Accessibility P1

Add a focus trap + aria-modal on mobile

The mobile scrim absorbs pointer events but doesn't intercept Tab. A keyboard user opens the sheet, the cursor lands on the primary button (good), they press Tab to explore — and walk out of the sheet into transcript marks they can't see. Screen readers face the same gap: the <aside role="complementary"> doesn't claim modal status, so the rest of the page stays in the AT tree.

Before — focus escapes

Current

Sheet · “Not useful” secondary
Sheet · close icon
⚠ Tab escapes here — into transcript marks behind the scrim
⚠ ...and into the bottom nav, hidden under the scrim

No aria-modal. No focus trap. Tab cycles into invisible page content — disorienting for keyboard and screen-reader users.

After — focus contained

Proposed
Sheet · primary button (focused on open)
Sheet · “Not useful” secondary
Sheet · close icon
↻ Tab loops back to the primary button — focus stays inside
Page content · removed from the AT tree via aria-modal="true"

Add role="dialog" + aria-modal="true" to the <aside> on mobile. Implement a Tab/Shift-Tab focus trap (small useEffect, ~25 lines, or pull in focus-trap-react). The page beneath drops out of the AT tree — screen readers correctly understand the sheet is the current focus context.

Desktop side-panel can keep the current behaviour (sheet accompanies the page, not blocks it) — apply trap + aria-modal only when the mobile scrim is present.

Minor observations (one-line polish, do once we're inside the file)

  • Shadow tokens: the sheet's shadow-[0_-18px_40px_-22px_rgba(0,0,0,0.22)] is a fixed rgba — doesn't follow the warm-cream / warm-navy split. Move to a token (--shadow-sheet-up / --shadow-sheet-side) tuned twice.
  • Naturalness sr-only repeat: HushStack line 63 emits <span class="sr-only"> — original</span> in the naturalness branch. The original is already in the visible <p> above. Screen readers hear it twice. Drop the span.
  • Drag handle redundancy: 32px width + 60% opacity is two reductions stacked. Pick one — at 32×4px it can carry full opacity without screaming.
  • Body spacing rhythm: space-y-6 (24px) sits between the brand's stated 32px (between sections) and 12px (within). Decide which rhythm wins — the answer is probably 32px between the hush stack / explanation / action region, and 12px within the hush stack itself.
  • Retry button context: error block's retry button has no aria-describedby pointing at the error message. Screen readers hear “Retry. Button.” — retry what? One attribute, 5 seconds.
Recommended shipping order: bundle the P1 trio (scrim/doc, border-t removal, focus trap) into one pass — they're the coherence + safety story. P2s can ship as a follow-up polish PR or be folded into the same pass if you want to close everything in one go. The minor observations slot in alongside whichever PR opens the relevant files.
