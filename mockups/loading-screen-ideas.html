Notes

NavProgress
The accent hairline at the top persists across all four — it's the primary "navigation in progress" signal. These concepts fill the content zone only, and live inside loading.tsx.
Dark mode
All four use only CSS custom properties. No dark-mode variants needed — the token swap at [data-theme="dark"] handles it automatically.
Reduced motion
A global @media (prefers-reduced-motion: reduce) rule clamps all durations to 0.01ms. A and D render their rest state cleanly. B renders static warm blocks. C renders nine static low-opacity bars — acceptable at-rest content hint.
Generic vs. page-specific
B gives users a structural preview even without page knowledge — it reduces the perceived jump when real rows appear. A, C, and D make no layout promises and work identically on every route.
Mix strategy
For routes that already have a tailored skeleton: swap the animate-pulse to a shimmer sweep (B pattern) and keep the page-specific shapes. For new routes where you don't want to build a skeleton: A or C. For the analysis/processing wait (already has PipelineStatus): D is redundant there; it fits routes like Settings or Write that load fast but occasionally stall.
