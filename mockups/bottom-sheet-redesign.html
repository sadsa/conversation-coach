What changes code

In DockedSheet.tsx the mobile anchor flips from bottom-[calc(4rem+env(safe-area-inset-bottom))] to bottom-0.

  • Footer absorbs the safe area — the sheet now touches the viewport edge, so the iOS home-indicator inset moves from the bottom nav onto the sheet's footer (pb-[max(1.25rem,env(safe-area-inset-bottom))] is already on the body; the new footer needs the same).
  • Mobile max-height can grow from 60vh / 75vh to ~78vh, since we just recovered ~64px of available height.
  • BottomNav stays mounted — no display:none, no remount on open/close. The sheet's z-index: 20 already paints over the nav's z-index: 30 ❶, so we bump the sheet to z-index: 40.

❶ Actually BottomNav is currently z-30 and the sheet is z-20, which means today the sheet would paint behind the nav if it ever overlapped — the only reason it doesn't is because of the bottom- offset. Anchoring change forces a z-index revisit either way.

Why this is right interaction

The nav is a "where am I, where can I go" affordance. While the sheet is open, the user is committed to this correction — nav targets are the wrong affordance to surface, and the gap above the nav today reads as a layout bug, not a deliberate frame.

  • Matches the iOS / Android system bottom-sheet pattern (sheets cover system chrome while open).
  • Primary action ("Save to my Study list") moves into the natural thumb zone instead of floating mid-screen.
  • Closes the visual seam between the sheet's bottom edge and the nav's top edge — two horizontal rules within ~10px of each other today.

What doesn't change scope

Everything outside the bottom anchor.

  • Desktop anchoring is untouched — sheet still slides in from the right, full-height, 400px wide.
  • No scrim. Brand context: sheets accompany, modals interrupt. Covering the nav doesn't make it modal — the rest of the underlying page is still tappable for outside-close.
  • Outside-tap dismiss still works — the document-level pointer handler in DockedSheet closes on any tap that isn't inside the sheet or matching preserveOutsideSelector. Tapping where the (covered) nav used to be will still register as inside the sheet and not dismiss.

Dark mode parity

Same Hush, same anchoring, same hierarchy.

The serif numerals and italic "you said" stay legible against warm navy because they carry their own weight — they aren't relying on a tertiary tone alone. Footer shelf is surface-elevated, one step up from the body — same lift the light version uses.

Same design language cohesion

Every type and chrome decision from mobile Hush carries over.

  • Floating controls. "3 of 12" in Source Serif, prev/next/close icon buttons. No header bar, no border-bottom — the controls sit inside the body padding.
  • Sentence-first body. "You said" eyebrow, italic struck-through "como soy", large serif "como me" as the answer. Slightly larger sizes than mobile (2.25rem vs 2rem on the answer) because we have the room.
  • Tinted footer shelf. surface-elevated background, no top border on mobile; on desktop we keep a single top border because the panel is taller and the footer needs to anchor.

What's different geometry

The mechanics adapt to the desktop frame; the voice doesn't.

  • No drag handle. Already true today — no swipe-down gesture to hint at on desktop. The mobile handle stays mobile-only (md:hidden).
  • Slides in from the right using the existing sheet-in-right keyframe. Bottom safe-area is irrelevant — the panel sits inside the browser window, not against an OS edge.
  • Footer sticky to the bottom of the panel via margin-top:auto inside the column flex layout. If the body is short, the footer still hugs the bottom — same anchor on every item.

One sub-decision desktop scrim

Mobile gets the soft scrim because the sheet replaces most of the viewport. Desktop is different: the right panel is meant to sit alongside the transcript so you can still read the source line while reviewing the correction.

  • Recommendation: no scrim on desktop. A dim on the left side would actively work against why we picked a side panel over a centered modal in the first place. The transcript is the source of truth — it should stay legible.
  • Rule we'd land on: scrim when the sheet REPLACES the page (mobile), no scrim when it sits ALONGSIDE (desktop).

Confirm and I'll wire it that way. Say the word and we'll add desktop scrim too if you want full parity.
