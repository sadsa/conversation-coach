# Impeccable Design Context

## Design Context

### Users
Primary user learning Rioplatense Spanish, plus a few friends. Used post-conversation — after recording a real Spanish conversation, users upload audio and review Claude's annotations during focused daytime study sessions. The workflow is: scan transcript → evaluate corrections → favorite valuable ones → track which phrases still need to be written on physical flashcards.

### Brand Personality
**Patient, Encouraging, Spacious**

A patient tutor that supports focused reading without rushing or overwhelming. The interface should feel calm and supportive, helping users evaluate corrections thoughtfully rather than gamifying progress or creating urgency.

### Aesthetic Direction
**Light mode primary.** Daytime use, reading-heavy workflow. Technical and utilitarian in execution — clarity over decoration.

**References:**
- Google Recorder: Clean transcript presentation, minimal chrome
- Gmail mobile: Immediately intuitive navigation and actions
- Jive World: Effective transcription display alongside audio context

**Anti-references:**
- Duolingo: No gamification, streaks, or playful mascots
- Generic admin dashboards: No dense data tables or utilitarian gray boxes
- Overly playful language apps: No celebration animations or emoji-heavy feedback

### Design Principles

1. **Readable first.** Large, comfortable text sizes. Generous line height. High contrast. The user has noted difficulty with small text — prioritize legibility over density.

2. **Scannable corrections.** Highlighted annotations must be immediately distinguishable during a quick scan. The eye should catch corrections without hunting.

3. **Spacious layout.** Breathing room between elements. Avoid cramped cards or dense information blocks. White space is a feature, not waste.

4. **Obvious actions.** Favoriting, tracking, and navigation should be self-evident. No hidden gestures or ambiguous icons.

5. **Informative without lecturing.** Explanations should be available but not forced. Let the user pull detail when they want it, not push it constantly.
