Voice control — placement

Header trigger that's always there, plus a WhatsApp-style status strip that pushes down from below the header while a session is active. Real app tokens, real proportions. Light theme; dark mirrors the same shape.

Mobile — three states

Idle is the resting position the user sees on every authenticated page. Connecting is <1s. Active expands a thin strip below the header with the audio dot, mute, and end. Tap mic again or X to dismiss.

What this gets right

One trigger surface across every authenticated route. No fight with the home UploadFab. No fight with BottomNav semantics. The active strip IS the "you're in a session" cue — no separate floating pill needed. Page content shifting down 44px when active is intentional: it's how the user knows the session is real, not a phantom WebSocket.

Open trade-offs

Header crowding. On a 320-wide phone the header now carries menu + back? + label + mic + theme. If we hit collisions, drop the section label while the strip is active — the strip already announces context.

Layout shift. Cheap mitigation: animate the strip in over 180ms with ease-out-quart; reduced-motion users get an instant snap. The main element gains a CSS variable --voice-strip-height that drives padding-top, so scroll position stays anchored to the user's reading position.

Discoverability. First-run cue: a small "Ask the coach anything" coachmark over the mic on first authenticated load. Same one-shot localStorage pattern as the existing UploadCoachmark.
